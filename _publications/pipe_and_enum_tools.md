## Pipe օպերատորը և Enum-ի գործիքակազմը (շարունակություն)

[Նախորդ հոդվածն](./enum_basics.md) ավարտվեց մի խնդրի լուծումով, որն աշխատում էր ճիշտ, սակայն ընթերցվում էր ներսից դուրս.

```elixir
Enum.sum(Enum.filter(Enum.map(1..10, fn x -> x * x end), fn x -> rem(x, 2) == 0 end))
```

Ընդամենը երեք գործողություն, և կոդն արդեն դժվար ընթեռնելի է։ Իրական ծրագրերում այդ գործողությունները հաճախ կարող են լինել հինգ, յոթ կամ ավել անգամներ։ Այս խնդրի համար Elixir-ն ունի հատուկ լուծում, որին այժմ կծանոթանանք։

### |> օպերատորը

Pipe օպերատորի (կարդացվում է «փայփ») կանոնը կարելի է ձևակերպել մեկ նախադասությամբ. ձախ կողմի արժեքը դառնում է աջ կողմի ֆունկցիայի **առաջին** արգումենտը։ Օրինակ `x |> f(a, b)` արտահայտությունը համարժեք է `f(x, a, b)`-ին։ ։ Այս պարզ օպերատորը սակայն էապես փոխում է կոդի տեսքն ու այն դարձնում չափազանց դյուրընթեռնելի.

```elixir
iex> 1..10 |> Enum.map(fn x -> x * x end) |> Enum.filter(fn x -> rem(x, 2) == 0 end) |> Enum.sum()
220
```

Արդյունքը կրկին 220 է, սակայն այժմ գործողություններն ընթերցվում են ձախից աջ, բնական հաջորդականությամբ. վերցնել 1-ից 10-ը, բարձրացնել քառակուսի, պահել միայն զույգերը, գումարել։ Ֆայլերում նման շղթաները սովորաբար գրվում են ուղղահայաց՝ ամեն գործողությունն առանձին տողում, և հենց այս տեսքն է բնորոշ Elixir-ի կոդին.

```elixir
1..10
|> Enum.map(fn x -> x * x end)
|> Enum.filter(fn x -> rem(x, 2) == 0 end)
|> Enum.sum()
```

Այժմ նաև պարզ է դառնում, թե ինչու է `Enum`-ի յուրաքանչյուր ֆունկցիա enumerable հավաքածուն ընդունում որպես **առաջին** արգումենտ. ամբողջ մոդուլը նախագծված է pipe օպերատորով հեշտությամբ աշխատելու համար։ 

Հարկ է հիշել մեկ կարևոր հանգամանք. pipe-ը միշտ լրացնում է **առաջին** արգումենտի տեղը։ Օրինակ՝ `5 |> Map.get(my_map)` արտահայտությունն իրականում կանչում է `Map.get(5, my_map)`, ինչը սխալ արդյունք կտա։

### any?, all?, find

Նախ նշենք մի պայմանավորվածություն, որին անուղղակիորեն հանդիպել ենք դեռ `member?`-ի դեպքում. Elixir-ում **`?` նշանով ավարտվող ֆունկցիաները միշտ վերադարձնում են `true` կամ `false`**։ Անվան մեջ առկա հարցական նշանն ուղղակիորեն հարց է ենթադրում։

`any?/2`-ը ստուգում է՝ արդյոք գոնե մեկ տարր բավարարում է պայմանին, `all?/2`-ը՝ արդյոք բոլորն են բավարարում.

```elixir
iex> scores = [85, 92, 78, 95, 60]
iex> Enum.any?(scores, fn s -> s >= 90 end)
true
iex> Enum.all?(scores, fn s -> s >= 70 end)
false
```

Այս ֆունկցիաների վարքի մեջ կա մի հետաքրքիր նրբություն. ի՞նչ են դրանք վերադարձնում դատարկ հավաքածուի դեպքում.

```elixir
iex> Enum.all?([], fn x -> x > 100 end)
true
iex> Enum.any?([], fn x -> x > 0 end)
false
```

`all?`-ը վերադարձնում է `true`, քանի որ դատարկ հավաքածուում չկա որևէ տարր, որը կխախտեր պայմանը։ `any?`-ը վերադարձնում է `false`, քանի որ չկա նաև որևէ տարր, որը կբավարարեր այն։ Առաջին հայացքից սա անսպասելի է, սակայն տրամաբանական տեսանկյունից սա միակ ճիշտ պատասխանն է։

`find/3`-ը վերադարձնում է պայմանին բավարարող **առաջին** տարրը, իսկ չգտնելու դեպքում՝ `nil` (կամ նշված default արժեքը՝ [ծանոթ](./modules_and_functions.md) `\\` մեխանիզմով).

```elixir
iex> Enum.find(scores, fn s -> s >= 90 end)
92
```

Կարևոր է `find`-ը չշփոթել `filter`-ի հետ. `filter`-ը վերադարձնում է **բոլոր** բավարարող տարրերը՝ list-ի տեսքով, իսկ `find`-ը՝ միայն առաջինը, հենց տարրը։ Գոյություն ունի նաև `find_index/2`-ը, որը տարրի փոխարեն վերադարձնում է նրա դիրքը։

### Տեսակավորում. sort և sort_by

```elixir
iex> Enum.sort([3, 1, 2])
[1, 2, 3]
iex> Enum.sort([3, 1, 2], :desc)
[3, 2, 1]
```

Առավել կիրառելին `sort_by/3`-ն է, որին փոխանցվում է ֆունկցիա՝ որը կիրառվում է enumerable հավաքածույի յուրաքանչյուր տարրի նկատմամբ, այսպիսով սահմանելով թե տվյալ տարրի հատկապես որ հատկանիշը հաշվի առնելով պետք է կատարվի տեսակավորումը։ Այստեղ նաև տեսնում ենք [`&` օպերատորի](./advanced_functions.md) գործնական կիրառությունը։

```elixir
iex> words = ["banana", "fig", "apple"]
iex> Enum.sort_by(words, &String.length/1)
["fig", "apple", "banana"]
```

Նույն խմբին են պատկանում `min/1`-ը, `max/1`-ը և դրանց `_by` տարբերակները.

```elixir
iex> Enum.max_by(words, &String.length/1)
"banana"
iex> Enum.min_max([3, 1, 2])
{1, 3}
```

`min_max`-ը մեկ գործողությամբ վերադարձնում է և՛ նվազագույնը, և՛ առավելագույնը՝ tuple-ի տեսքով։ Դատարկ հավաքածուի դեպքում այս ֆունկցիաները սխալ են առաջացնում. այդ դեպքի համար լրացուցիչ արժեք սահմանելու տարբերակները կարելի է գտնել `h Enum.max/3`-ի միջոցով՝ [մեզ ծանոթ գործիքով](./anonymous_functions.md)։ `Enum.reverse/1`-ը, որը մեզ ծանոթ է դեռ list-երի հոդվածից, պարզապես շրջում է տարրերի հերթականությունը՝ առանց տեսակավորելու։

### uniq, frequencies, group_by

`uniq/1`-ը հեռացնում է կրկնվող տարրերը՝ պահելով առաջին հանդիպածը.

```elixir
iex> Enum.uniq([1, 2, 2, 3, 1])
[1, 2, 3]
```

Հաջորդ երկու ֆունկցիաները բացառություն են կազմում առաջին հոդվածում նշված «`Enum`-ը վերադարձնում է list» կանոնից. դրանք վերադարձնում են [map](./maps.md)։ `frequencies/1`-ը հաշվում է, թե ամեն տարր քանի անգամ է հանդիպում.

```elixir
iex> Enum.frequencies(["ant", "bee", "ant", "ant", "bee"])
%{"ant" => 3, "bee" => 2}
```

`group_by/2`-ը տարրերը բաժանում է խմբերի՝ ըստ նշված ֆունկցիայի վերադարձրած արժեքի.

```elixir
iex> Enum.group_by(["ant", "buffalo", "cat", "dingo"], &String.length/1)
%{3 => ["ant", "cat"], 5 => ["dingo"], 7 => ["buffalo"]}
```

Այժմ միավորենք մի քանի գործիք մեկ շղթայում՝ pipe, frequencies, max_by և tuple-ի pattern matching ֆունկցիայի արգումենտում.

```elixir
iex> ["red", "green", "red", "blue", "red"] |> Enum.frequencies() |> Enum.max_by(fn {_word, count} -> count end)
{"red", 3}
```

Ընդամենը մեկ տողով որոշեցինք ամենահաճախ հանդիպող բառը և դրա քանակը։ Այս տողում միաժամանակ կիրառվում է վերջին մի քանի հոդվածում ուսումնասիրվածը։

### take, drop, at

```elixir
iex> Enum.take(1..10, 3)
[1, 2, 3]
iex> Enum.take(1..10, -2)
[9, 10]
iex> Enum.drop(1..10, 7)
[8, 9, 10]
```

`take`-ը վերցնում է տարրերը սկզբից (բացասական թվի դեպքում՝ վերջից), `drop`-ը հեռացնում է սկզբից և վերադարձնում մնացածը։ Գոյություն ունեն նաև պայմանի ստուգումով տարբերակները. `take_while/2`-ը վերցնում է տարրերն այնքան ժամանակ, քանի դեռ ֆունկցիայի պայմանը բավարարվում է, և կանգ է առնում առաջին չբավարարվող տարրի վրա.

```elixir
iex> Enum.take_while([1, 2, 5, 1], fn x -> x < 3 end)
[1, 2]
```

Ուշադրություն դարձնենք, որ վերջին 1-ը չվերցվեց, քանի որ ֆունկցիան արդեն կանգ էր առել 5-ի վրա։ `drop_while/2`-ը նույն սկզբունքով հեռացնում է սկզբի տարրերը։

`at/3`-ը վերադարձնում է նշված դիրքի տարրը՝ 0-ից սկսվող համարակալմամբ, չգտնելու դեպքում՝ `nil` կամ նշված default արժեքը, իսկ `-1`-ի դեպքում՝ վերջին տարրը.

```elixir
iex> Enum.at([:a, :b, :c], 0)
:a
iex> Enum.at([:a, :b, :c], 5)
nil
```

Հիշենք list-երի հոդվածից, որ list-ում կամայական դիրքի տարրին հասնելը O(n) բարդության գործողություն է, ուստի `at`-ը կրկնվող կանչերում կիրառելը արդյունավետ չէ։ Երբ տարրերին դիրքով դիմելը հաճախակի անհրաժեշտություն է, նպատակահարմար է կիրառել tuple կամ map։

### Փորձեք ինքներդ

Բացեք iex-ը։ Սկզբի համար [Նախորդ հոդվածի](./enum_basics.md) եզրափակիչ խնդիրը վերաշարադրեք pipe-երով և ստուգեք, որ արդյունքը կրկին 220 է։ Այնուհետև վերցրեք `scores = [85, 92, 78, 95, 60]`-ը և առանձին տողերով պատասխանեք հետևյալ հարցերին. արդյոք բոլո՞րն են 50-ից բարձր, ո՞րն է 80-ից ցածր առաջին գնահատականը, որո՞նք են երեք ամենաբարձրերը (հուշում՝ `sort`՝ `:desc`-ով, ապա `take`)։ Ապա որևէ անունների list-ի վրա կիրառեք `frequencies`-ը, իսկ `group_by`-ի միջոցով խմբավորեք անունները ըստ առաջին տառի՝ օգտագործելով`&String.first/1` ֆունկցիան։ Բոլոր առաջադրանքները փորձեք լուծել pipe-երով։

### Ի՞նչ է սպասվում առջևում

Մեր գործիքակազմն արդեն բավականին ընդլայնվել է, սակայն ամենակարևոր ֆունկցիան դեռ չենք ուսումնասիրել։ Հաջորդ՝ եզրափակիչ հոդվածում կծանոթանանք `reduce`-ին՝ ֆունկցիային, որի վրա կարելի է կառուցել `Enum`-ի գրեթե բոլոր մյուս ֆունկցիաները։
